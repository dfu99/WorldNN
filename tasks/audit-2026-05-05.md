## 3. Single biggest blocker

**The official `neurips_2026.sty` has not been downloaded.** The submission
will use a placeholder stub (`Placeholder NeurIPS 2026 style file. Replace
with the official neurips_2026.sty from https://neurips.cc before
submission.`).

### Why hasn't it been resolved?
Three contributing factors:
1. The placeholder is functional enough for `pdflatex` to compile cleanly
   (10-page PDF generated successfully in this audit). The visual output
   is non-conformant but not a hard error, so it never blocked downstream
   work.
2. Recent sessions were spent on AFK research sweeps (iter-2, iter-3) and
   compliance/style cleanups, not the submission mechanics.
3. The pod has been unreachable since 2026-05-01, eating attention that
   would otherwise have gone to deadline mechanics.

### Severity
**Critical for May 6.** A submission with the wrong style file is at risk
of desk-rejection. It is also a 5-minute fix:
```
curl -L https://neurips.cc/Conferences/2026/PaperInformation/neurips_2026.sty \
  -o paper/neurips2026/neurips_2026.sty
cd paper/neurips2026 && pdflatex main.tex
```
(The exact URL depends on the conference page; latex2e archive of
neurips_2026.sty also works.)

## 8. Code audit (post-primary-audit follow-up)

### What's brittle
1. **No checkpoint resumption granularity in obj-027/028.** Scripts only
   write every 5 configs. Halulujah preemption at config 2 lost work.
   Fix: write checkpoint after every config, since IO is cheap.
2. **`compute_sa_sensory` re-uses target=(0.8,0.8) hard-coded** despite
   parameterization elsewhere. Ok for current usage; brittle if target
   varies.
3. **`mc runpod` shell helpers depend on stale ssh state.** Last 4 days
   showed this (SSH refused even when `mc runpod check` returned data).
   Not WorldNN-specific but bites us.
4. **`Organism` ignores `action_dim` for 1D tasks.** obj-028 hacks around
   it by reshaping `action_mean[:, :1]`. A clean fix would parameterize
   the policy head.

### Tests that don't exist
- `tests/test_components.py` covers RockPushMatter and World step. 29
  tests pass.
- **No tests for** `Organism` shape contracts, `MultiRockMatter`,
  `ContinuousMatter`, `EnvironmentVAE`, the SA computation, or the
  KSG-MI estimator. Numerical drift in any of these would silently
  invalidate paper claims.
- **No tests for** the sensory-dim slicing path
  (`obs = emission[:, :sensory_dim]`). A test that confirms the slice
  preserves expected shape and dtype across `sensory_dim ∈ {2,4,8,16}`
  would catch regressions cheaply.

### Dead/orphaned code
- `experiments/run_*.sbatch` — already scrubbed in commit `f14b085`.
- `experiments/__pycache__/` — already gitignore-able, not committed.
- `tasks/queue.yaml.broken` — backup file in root, never tracked, never
  cleaned. Decide once: delete or move to a `tasks/_backups/`.
- `experiments/sa_proxy.py` — exists, used in obj-022, fine.
- Several `experiments/obj025_*.py` analysis scripts have overlap (e.g.,
  `obj025_claim_evidence_summary.py` is a one-shot table renderer).
  Acceptable for one-off scripts; do not refactor.
